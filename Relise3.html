Dúvidas sobre configuração
  • 1). Como configurar uma política de preço diferenciada para um grupo de CNPJs?

    A configuração deve ser realizada dentro do ERP/Integrador e deverá ser enviada via integração para a plataforma Infrashop.

    Podendo seguir o exemplo do manual:

    3.3 Política DEFAULT: Essa política se aplicará para todos os clientes que não fazem parte de uma política do tipo cluster, não havendo necessidade de incluir CNPJ’s, uma vez que a plataforma identifica o nome DEFAULT para esses clientes.

    3.4.1 Política Cluster O distribuidor pode enviar outra política diferente da Default para os clientes que praticarem preços diferenciados em comparação a primeira política, importante lembrar que essa política precisa conter a lista de CNPJ’s declarados, todos os produtos e tabelas de preços que serão comercializados.

    Para mais detalhes verificar o Manual de Integração B2B:

    https://developers.infracommerce.com.br/docs/middleware/pry5qdf6ha7zy-1-manual-de-integracoes-b2-b

  • 2). Como automatizar o envio massivo do limite de crédito para quando um CNPJ for cadastrada ele já ter limite disponível?

    A plataforma possibilita receber os limites de crédito de forma antecipada, mesmo que o cliente não esteja cadastrado. Deve-se utilizar as rotinas necessárias do ERP para realizar o esse envio a fim de possibilitar essa antecipação.

  • 3). Como configurar diferentes políticas de boleto, como boleto escalonado ou prazos diferenciados para determinados grupos de CNPJ de acordo com a política de pagamentos do próprio distribuidor?

    Para a realização desse processo deve ser solicitado o cadastro dos prazos de pagamento.

    Por exemplo:

    Para boleto faturado de 25 dias, solicitar o cadastro de CRL-25.

    Para boleto faturado parcelado em 3 parcelas para 10, 20 e 30 dias, solicitado o cadastro CRL-10_20_30.

    Além da solicitação de configuração de preço na plataforma, é necessário o envio de preços para essa novos prazos de pagamento para todos os produtos (SKU`s) dentro da política de preço que ela for ser utilizada.

  • 4). O que é o período de venda assistida? Como melhor utilizá-lo para tirar dúvidas no processo de integração?

    Venda assistida é diferente de operação assistida.

    Venda assistida: É a venda feita através de um representante do Seller utilizando a nossa plataforma de vendas.

    Operação assistida: É o período de acompanhamento da operação de 10 dias úteis após o Golive do Rollout (Testes), onde o Seller dentro do grupo técnico poderá sanar suas dúvidas e obter suporte do time técnico da infracommerce sem a necessidade de abertura de chamado via canais de NRP.

  • 5). Alguns produtos não estão aparecendo no site. Como verificar se todos os pickings e EANs que o distribuidor trabalha estão ativos na plataforma? Como corrigir esse problema?

    O primeiro envio é feito acompanhado junto ao analista de testes da Infracommerce, o qual irá analisar e verificar se o produto integrou corretamente, validando se o produto integrou e se o mesmo está aparecendo no front da loja.

    Após o periodo de operação assistida, caso sinta falta do produto deve ser realizado sempre o envio de uma nova carga de estoque e preços completos. Após envio, é possível validar os valores integrados na plataforma, acessando como cliente e validando os preços dos produtos no front end da loja.

    Com relação ao Picking (maior do que 1 unidade), Caso esse produto não possua cadastro na plataforma, o mesmo deve ser solicitado para realização de cadastro.

    Caso tenha a necessidade de uma análise mais detalhada o seller deve realizar a abertura de um chamado via equipe de NRP.

  • 6). Como integrar o pedido com o código do vendedor do distribuidor para garantir acuracidade na informação sem intervenção manual posteriormente?

    Os pedidos de venda assistida são enviados com duas informações: Nome do vendedor e ID do vendedor, caso essas informações tenham alguma especificidade ( Exemplo: Código do vendedor deve ser CPF) isso deve ser alinhado com o time comercial da infracommerce para os cadastros serem realizados dentro da forma esperada.

  • 7). Quando um cliente é cadastrado, ele passa por uma aprovação interna do distribuidor. Para onde essa verificação é direcionada? Como automatizá-la em caso de cliente já cadastrado no sistema do distribuidor?

    A tratativa de aprovação ou reprovação e disponibilização ou não de crédito para o cliente é de total responsabilidade do Seller. Todo cliente cadastrado na plataforma infracommerce é enviado ao ERP do distribuidor e caso tenha a necessidade de aprovação automática o mesmo deve realizar uma customização de rotinas.

  • 8). Como configurar para que o envio das tracking de pedido sejam enviadas automaticamente? FAT, ENT, etc...

    Toda e qualquer automatização de processo deve fazer parte de uma possível customização do ambiente do Seller.

  • 9). É possível o próprio distribuidor criar alguma promoção e subir no Reppos?

    Não. Toda lista de promoções criadas, advém de solicitação da indústria para o time Infracommerce.

  • 10). O que é necessário enviar para que o produto seja desativado no site?

    Existem duas formas, sugerimos que as duas sejam feitas:

    • Enviar estoque 0(zero) para o produto.
    • Enviar preço 0 (zero) para o produto.

    Para voltar a comercializar esse produto, basta realizar o envio de estoque e preço com valor maior que deseja.

  • 11). Como faço para mover um cliente para uma nova política? É necessário enviar todos os preços novamente?

    Utilizando o exemplo 3.5 do manual (Inclusão de clientes para uma política de preço), neste exemplo é enviado 3 clientes para a política desejada.

    Não há necessidade de enviar produtos e preços novamente, a plataforma sempre armazena a última requisição enviada e recomendamos que atualizações de novos clientes, produtos e tabelas de preços sejam realizadas conforme a necessidade, esse processo também otimiza a fila o tempo de integração.

  • 12). Os clientes que já tenho na minha base já possuem cadastro da plataforma infracommerce?

    Pode ser que o cliente de seu ERP já tenha cadastro em nossa plataforma, porém temos que ressaltar que todos os clientes da plataforma infracommerce é enviado para o seu ambiente, Tendo ciência que isso possa ocorrer, existe a possibilidade dele ser enviado novamente (podendo ter o cadastro sobrescrito).

    OBS: Caso os dados do cliente não devam ser sobrescritos, o distribuidor deverá realizar uma tratativa (Parametrização) para que os dados não sejam atualizados.

  • 13). Posso enviar uma NFS mais de uma vez?

    Um pedido pode receber várias notas fiscais.

    Exemplo:
    Caso um pedido possua itens bonificados, o distribuidor pode enviar duas notas fiscais, sendo uma para os itens cobrados e outra para os ítens bonificados.

    Importante: Após enviar a tracking de FAT nenhuma nota fiscal poderá ser integrada, uma vez que ela indica o encerramento do faturamento.

  • 14). Posso faturar um pedido cancelado?

    Não, um pedido cancelado encerra completamente o fluxo de novas trackings.

  • 15). Como fazer com que um cliente não possa efetuar pedidos na plataforma?

    Para que um cliente não possa efetuar pedido na plataforma é necessário enviar o status RPV (Reprovado) via integração, assim ele não conseguirá efetuar nenhum tipo de compra na plataforma.

    OBS: Caso queira habilitar esse cliente para realizar compras novamente é necessário enviar o status APV (aprovado).

Dúvidas sobre erros frequentes
  • 1). Está aparecendo a mensagem de erro "Sua área ainda não é atendida por nenhum distribuidor" para alguns CNPJs, o que fazer nesse caso? Como corrigir?

    Solicitar ao nosso suporte a validação do cadastro de área de atendimento.

  • 2). Preciso cadastrar um vendedor novo no venda assistida? O que eu faço?

    Abrir solicitação ao nosso suporte, para que nosso time de CS efetue o cadastro do vendedor.

  • 3). Um pedido está demorando muito para integrar, o que fazer?

    Caso o período médio de 4 horas tenha sido excedido, será necessário abrir um chamado ao nosso time de NRP.

  • 4). O arquivo de tracking foi enviado de modo errado por algum motivo. O que fazer agora? Como corrigi-lo?

    Envio da tracking NFS com erro pode ser corrigido com um novo envio, desde que não tenha ocorrido a integração da tracking FAT. Após a integração de FAT, não é mais possível a integração de qualquer outra NFS. Caso necessário, deve ser aberto um chamado ao nosso time de NRP.

  • 5). Por que não consigo efetuar um pedido de boleto faturado sendo que o cliente possui limite de crédito ou status APV? (nunca as duas informações).

    Sempre tentar efetuar um novo envio de limite para esse cliente e caso não apareça verificar se o distribuidor efetuou o envio de preços para os meios de pagamentos que ele deseja efetuar o pedido.

    Se mesmo assim não estiver disponível deve ser verificado se os tipos de pagamento estão habilitados para o distribuidor (Essa verificação deve ser realizada via NRP).

  • 6). Porque o cliente está visualizando os preços errados ou de outra política de preço?

    Sempre deve ser verificado os preços enviados para a plataforma e verificar se esse cliente está atribuído a uma política que não deveria, seja por uma falha no envio, ou outro motivo, mas é possível corrigir enviando uma requisição de preços para plataforma que faça a inclusão do cliente para a política desejada. Utilize o exemplo 3.5 do manual de integração B2B (Exemplo de inclusão de clientes a uma política) para essa tarefa.

  • 7). É possível excluir alguma requisição enviada erroneamente?

    Não, as requisições não podem ser excluídas, mas muitas podem ser corrigidas.

    Exemplo:
    Envio de estoque incorreto, o que fazer? enviar uma nova requisição com o valor correto, ele sobrescreverá o valor enviado anteriormente.

Dúvidas sobre vendedores
  • 1). Ao selecionar um determinado CNPJ não aparece nenhum produto no site. Que erro é esse e como corrigi-lo?

    Verificar se o cliente consta com status reprovado, caso constar, enviar status de aprovação, outra possibilidade é validar se esse cliente está inserido dentro de uma política de preços que contenha os produtos, o preço e estoque destes produtos devem ser maiores do que zero para que o mesmo apareça no site.

    Caso o erro persistir após a verificação dos processos acima, entrar em contato com nosso time de NRP.

  • 2). Porque não aparece opção de boleto parcelado para o cliente? Como corrigir esse problema?

    Solicitar ao nosso suporte a validação das políticas de preços integradas para verificar se a política da qual o cliente se encontra o qualifica para o prazo de pagamento que ele deseja utilizar.

    Vale ressaltar que, as tabelas de preços enviadas parceladas: exemplo: 7-14_21_28_35 também devem estar habilitadas do lado da plataforma caso decidam trabalhar com outros métodos parcelados que não eram praticados até o momento, essa ativação é feita através de uma solicitação para o time de NRP, que, após habilitado, os preços para essas novas tabelas devem ser enviados novamente.

  • 3). O que fazer caso o status do pedido não atualize?

    Solicitar ao nosso suporte (NRP) a validação da integração do status do pedido na plataforma.

  • 4). Eu preciso de mais limite de crédito para o meu cliente, o que eu faço?

    Basta enviar o valor de limite de crédito desejado para o cliente via integração. Nossa plataforma sobrescreve a informação anterior a partir da informação do novo valor enviado.

  • 5). O preço do tablet está diferente do preço do Reppos, o que fazer?

    Você deve entrar em contato com o distribuidor para verificar e validar a tabela de preços enviado para o site Reppos.

  • 6). O preço de uma promoção não está carimbando no meu carrinho, o que fazer?

    Você deve entrar em contato com o nosso time de NRP (Núcleo de relacionamento com o parceiro) e enviar as evidências para ter o suporte, os contatos são:

    email: atendimentonrp@sacreppos.com.br / Whatsapp: 11 95318-0983.

Dúvidas gerais
  • 1). Como desativar um produto no e-commerce?

    Para inativar um produto na plataforma é esperado estoque ou preço zero, assim o produto ficará indisponível para vendas. Para ativar esse produto novamente basta enviar preço e estoque positivo assim o mesmo ficará novamente disponível na plataforma.

  • 2). Se quisermos bloquear a venda nas formas de pagamento CRL-7,14,21,28, e deixar somente BILL e CREDIT_CARD ?

    Se o seller enviar preço zerado para CRL7,14 e asssim por diante, inativa a visualizacao para compra de boleto faturado. Se mandar preço zerado para o DFL, indisponibiliza tudo, pois DFL é o critério obrigatório para o produto aparecer no site.

  • Pedido faturado ou faturar pedido?

    Pedido faturado é o boleto a prazo, faturar pedido é referente ao processo que o distribuidor faz de todo o fluxo para emitir uma NF-e (Nota fiscal eletrônica).

  • 3). Por que meu cliente não pode comprar a prazo?

    Para que um cliente possa comprar a prazo é necessário que o distribuidor envie um limite de crédito para o cliente, assim as condições a prazo serão habilitadas.

  • 4). Como inativar um cliente na loja?

    Para que um determinado cliente não possa comprar na loja é necessário o envio de status do cliente como “RPV” (Reprovado), assim o cliente fica sem visualizar as opções de produtos disponibilizadas pelo distribuidor. Caso queira que esse cliente volte a comprar basta atualizar o status do cliente para APV (Aprovado).

  • 5). Enviei limite de crédito para o cliente, mas ele não pode comprar a prazo!

    Não é possível. A plataforma entende que qualquer limite de crédito recebido é apto a ter condições a prazo disponibilizadas pelo distribuidor que realizou o envio de crédito para o cliente.

  • 6). Posso ter preços diferenciados por cliente?

    Sim, existe a politica de preço cluster para essa ação, onde é enviado nesta política os CNPJ dos clientes que terão aquelas condições diferenciadas de pagamento e preço.

  • 7). Funcionamento das políticas de preço.

    Possibilita trabalhar com múltiplos preços para os clientes (Consumidor final) através do envio de políticas de preços distintas, como por exemplo:

    Distribuidor envia uma política default, essa será aplicada para todos os clientes como padrão, e pode enviar uma segunda política com preços diferentes da default atribuindo para 1 ou N clientes.

  • 8). O que seria o faturamento parcial?

    Faturamento parcial é o faturamento de uma quantidade menor que a quantidade comprada pelo cliente.

    Exemplo:
    Um pedido foi realizado com 10 unidades de um produto, porém, o distribuidor possui apenas 9 unidades em seu estoque, para não perder essa venda é possível faturar os 9 itens para o consumidor.

    Ponto de atenção: O envio da nota fiscal (NFS) para a plataforma deve constar 9 itens faturados.

  • 9). Como identificar uma promoção (desconto) aplicado em um produto na requisição de pedido?

    No pedido enviado via-plataforma são descritas as características e informações pertinentes ao desconto.

    Exemplo:

    Tag totalDiscountAmount - que informa o total de descontos que o pedido possui.

    Tag promotionList - lista de informações pertinentes ao produto que está com promoção.

    Assim como também pode ser verificado no nome da promoção, tipo de promoção etc. Toda particularidade pertinente ao tipo de pedido sempre será enviada nos campos do XML. É essencial que o ERP esteja tratado para distinguir essas informações, por este motivo é muito importante tratá-las na etapa de testes.

  • 10). O que seria o kit bonificado?

    É um pedido que foi realizado no modelo de combo.

    Exemplo:
    Compre 1 e leve 2. Nesse caso um dos itens deve ser bonificado, sendo faturado com o valor igual a zero na nota (Nota Bonificada) que é enviada para a plataforma.

    O distribuidor também deve informar na nota o código do kitSkuId, que deve ser identificado no XML do pedido.

    Vale ressaltar que um produto do tipo bonificado é identificado no pedido através da tag “skuType”. Podemos identificar quando o texto neste campo for igual a FFR, assim podemos saber quando o produto é bonificado ou não.

    OBS: Os demais itens que não forem BONIFICADOS dentro do pedido, devem ser faturados normalmente dentro do pedido.

  • 11). O que é o reprocessamento das faixas de CEP?

    É o envio massivo de clientes da plataforma infracommerce para o ERP do seller que pertencem a área de atendimento do distribuidor.

    Exemplo: Distribuidor atende o range de faixa de CEP 1000000 a 200000, esses clientes que foram cadastrados anteriormente na plataforma Infracommerce serão enviados via requisição para o seller.

    Importante: Caso o distribuidor já possua o cadastro desses clientes em seu ERP e deseja que os dados cadastrais não sejam alterados, será necessário fazer essa tratativa dentro da integração/ERP para os dados que não desejam ser atualizados.

  • 12). Posso faturar um pedido feito com produtos com caixa, faturando em unidade?

    Não. Produtos em unidade e caixa são gerenciados de forma distinta na plataforma, portanto o envio da nota fiscal devem ser realizados com as SKU`s da mesma forma que são enviados no pedido. (Se for unitário, faturar o unitário, se for caixa faturar a caixa). Caso esses ítens não estejam em conformidade, haverá divergências em repasses e ruptura do pedido.
