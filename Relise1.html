Introdução

Implantação B2B é o processo de testes entre o distribuidor e a plataforma Infra.Shop, realizando validação de requisições de preço, estoque, cliente, status, pedidos, pagamentos e trackings, garantindo a integridade de dados, o fluxo de testes é dividido em duas etapas a de homologação (HCP) e produção (PRD).

Por padrão, antes do início das integrações, a equipe de PMOs responsáveis realizam uma reunião apresentando o projeto (Kickoff), mostrando os prazos, como iniciar, e respondendo as dúvidas que possam aparecer no momento. O PMO tambem é responsavel por criar o ticket de premissas (necessario para configuração do setup do distribuidor), durante todo o processo por padrão é aberto o primeiro ticket de premissas para configuração do ambiente de homologação, após está configuração é iniciado os testes.

Após o termino dos testes em homologação é solicitado um 2º ticket (virada) para o ambiente de produção.

Etapas de Implantação

1 - Acompanhamento e desenvolvimento


Acompanhamento

Os testes são feitos sempre em conjunto, Distribuidor - Infracommerce, e durante toda a etapa de Rollout o distribuidor terá um analista disponível via Skype ou Chat (Ferramenta de Google) durante todo horário comercial, de segunda a sexta feira.

E nesse período serão realizados os cenários apontados pelo analista, assim como também podem ser solucionadas duvidas do processo. Terá também no grupo o PMO do projeto para apoiar em qualquer eventualidade.

Caso ocorra que o Analista Infracommerce, responsável pelo projeto, não esteja disponível, será alocado outro analista para auxiliar e dar continuidade aos testes. Durante toda a fase de testes, desde as validações iniciais até a operação assistida (Go Live), sempre terá um analista para acompanhar o processo.

A infracommerce sempre se compromete com a excelência no atendimento de nossos distribuidores.

Desenvolvimento

Após a reunião de apresentação do projeto, e o envio do arquivo zip com os dados necessários, o distribuidor deve efetuar o desenvolvimento dos três Endpoints, sendo eles: CUSTOMER, ORDER e PAYMENT, necessários para a comunicação entre plataforma e ERP.

O desenvolvimento dos três endpoints publicados em ambiente do distribuidor, são os pré-requisitos que solicitamos para liberarmos as credenciais da infracommerce, responsável para os envios de stock, preço, status de cliente, limite de crédito e tracking. É de extrema importância que o distribuidor desenvolva esses tópicos para que no início dos testes sejam fluídos.

Seguindo com o fluxo de integração, com os endpoints desenvolvidos pelo distribuidor (Customer, Order e Payment) e disponibilizado à equipe de testes, é criado o setup da loja para inicio dos testes em homologação, após essa configuração a equipe de testes disponibiliza os endpoints (Stock, Pricing, Customer, Tracking, Voucher) para o distribuidor também configura em seu ambiente.

Após o desenvolvimento é realizado a etapa de validação de endpoints.


É efetuada uma validação nos endpoints de Cliente(Customer), Pedido(Order) e Pagamento(Payment) do Seller. Com os retornos corretos, é solicitado à equipe de projetos a emissão do ticket de premissas para homologação.

Após a validação dos endpoints solicitamos o ticket de premissas de HCP para a equipe de projetos. Com o ticket de premissas emitido, iniciamos as configurações do ambiente.

Saiba mais sobre o desenvolvimento no manual de integração B2B.

2 - Etapas de testes em homologação


O Ambiente de homologação (HCP) é configurado pela equipe de testes e integrações, após a disponibilização do ticket de premissas. Estando o setup (Loja) já configurado é passado os endpoints para o distribuidor configurar em seu ambiente, após a confirmação do distribuidor, é iniciado as etapas de testes em ambinete de homologação, são elas :

Obs: Vale ressaltar que as requisições e respostas devem está de acordo com o manual de integração.

Pré Rollout


É um resumo dos testes completos de HCP a fim de validar todos os endpoints e tipos de integração que serão realizadas no decorrer da homologação.


Envio de todo o estoque que irá utilizar em ambiente de homologação, recomendado 3 SKU´s unitarios e 3 SKU´s caixa.


Envio de politica de preço default dos mesmos sku´s que foram utilizados no envio de estoque.


Envio de um cliente para o ambiente do distribuidor.


Envio do primeiro limite de crédito.


Envio do primeiro pedido em condição boleto a vista, enviando aprovação de pagamento (AP), faturado por completo.


Mudar status do cliente pendente (PEN) para reprovado (RPV).


Enviar atualização de politica de preços default.


Cliente e limite de crédito


Solicitado o envio de atualização de status de cliente e limite de crédito, alem da loja enviar atualização do cadastro de cliente.


Envia atualização de status passando o cliente de reprovado (RPV) para aprovado (APV), com limite de crédito no valor de R$ 5.000,00.


Envia atualização de cadastro de cliente, alterando o numero de telefone.


Pedidos com politica de preço default.


Realizado diversos cenários de pedidos a fim de abranger todas as possibilidades de pedido real em produção. Utilizando preços da politica default.


Envia pedido boleto a vista, com aprovação ou reprovação de pagamento.


Envia pedido com cartão de crédito, com aprovação ou reprovação de pagamento.


Envia pedido com pagamento PIX, com aprovação ou reprovação de pagamento.


Pedido já integrado.


Envio de pedido com SKU CX pago via vale troca e faturado por completo.

Preço


Solicitado o envio de politica de preços cluster, com informação de CNPJ do cliente que terá preço diferente da politica default.


Envia preços da politica cluster diferentes dos preços integrados para a política DEFAULT. Deve enviar nesta política cluster o cliente cadastrado no pré-rollout.

Pedidos com politica cluster


Realizado diversos cenários de pedidos a fim de abranger todas as possibilidades de pedido real em produção. Utilizando preços da politica cluster.


Envia Pedido boleto a vista, reprovado e cancelado.


Pedido cartão de crédito, reprovado e cancelado.


Pedido boleto 21 dias, reprovado e cancelado.


Pedido boleto 28 dias, faturado parcialmente.


Pedido boleto faturado, kit promocional, faturado por completo.


Pedido boleto a vista + vale troca, aprovado e faturado por completo.


Dois pedidos com pagamento PIX.


Pedido pago com PIX + Vale troca.

Atualizar politica de preço


Solicitado o envio de atualização da politica de preços cluster alem de mover um cliente da politica cluster para politica default.


Envia atualização de preço da politica cluster.


Envio da política default com o CNPJ do cliente presente na política CLUSTER..

3 - Virada para produção


Ao término dos testes de homologação é solicitado os endpoints de produção que serão utilizados pelo Seller, com os endpoints em mãos, é solicitado a abertura do ticket de premissas de PRD à equipe de projetos.

Efetuada a configuração em ambiente de produção com os endpoints e credenciais de ambiente produção do Selle presentes no ticket de premissas de PRD.

Cenários reduzidos de testes abrangendo envio de Estoque, preços, Cadastro de cliente, envio de Status APV, Limite de crédito, emissão de pedido que logo deve ser cancelado e envio massivo de clientes já cadastrados na plataforma que estejam dentro do atendimento por faixa de cep configurados para o Seller(etapa necessária para novos Seller’s na plataforma e não necessário para homologação de melhorias de clientes já em produção).

4. Operação assistida (GO Live).


Após o termino dos testes em produção (PRD) é aprovação por parte da indústria e PMO é iniciado a operação assistida geralmente tendo um período de 10 dias corridos, realizando o acompanhamento do fluxo de integrações e relatório diário sobre a quantidade de requisições.

Tipos de Integração

Integração padrão


Essa integração consiste em todos os cenários de homologação, desde envio de estoque e preço, cadastro de clientes, envio e remoção de limite de crédito, envio de todas as formas de pedidos em todas as modalidades de pagamento, e envios de tracking. Em resumo, iremos a testar o fluxo de vida completo de vários tipos de pedido.


Para execução deste tipo de integração é necessario os testes de: 

- Desenvolvimento
- Pré Rollout
- Cliente e limite de crédito
- Pedidos com politica de preço DEFAULT
- Preço
- Pedidos com politica de preço CLUSTER
- Atualizar politica de preço
- Virada para produção
- Operação Assistida (Go Live)

Integração padrão + LTM


É um meio de pagamento apenas disponível para as Lojas Compra Agora, também chamado Vale Compra Agora, e que está mais detalhado no Manual de integrações. Assim como os testes de Pricing, existe a possibilidade de realizar testes apenas com LTM, caso o distribuidor precise implementar essa forma de pagamento. Os testes se referem a integração de pagamentos de pedidos via LTM, onde serão realizados diversos pedidos junto com outras formas de pagamento. Exemplo: LTM + Cartão de Crédito.


Para execução deste tipo de integração é necessario os testes de: 

- Desenvolvimento
- Pré Rollout
- Cliente e limite de crédito
- Pedidos com politica de preço DEFAULT
- Preço
- Pedidos com politica de preço CLUSTER
- Peidos pagos via LTM
- Atualizar politica de preço
- Virada para produção
- Operação Assistida (Go Live)

Integração de melhoria Pricing


Pricing se refere ao envio das Políticas de Preço, com a possibilidade de valores diferenciados por CNPJ de cliente. Caso um distribuidor já se encontre em produção com o modelo de preços convencional, e queira implementar o modulo Pricing, será necessário refazer os testes de homologação apenas considerando os cenários de preço e pedidos.


Para execução deste tipo de integração é necessario os testes de: 

- Cliente e limite de crédito
- Pedidos com politica de preço DEFAULT
- Preço
- Pedidos com politica de preço CLUSTER
- Atualizar politica de preço
- Virada para produção

Duração dos Testes

O tempo estimado para finalização de todos os cenários de testes em homologação e produção no Rollout são de 30 dias.

E adicionalmente teremos outros 10 dias de acompanhamento na Operação Assistida.

Aguardando o distribuidor validar e integrar os dados. Anexar evidencias no Monday .

Atividades Tempo medio de execução
